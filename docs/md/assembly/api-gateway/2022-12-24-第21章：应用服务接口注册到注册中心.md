## 二、服务注册

每一个做为提供HTTP接口的RPC应用服务，都需要基于引入的SDK组件，采集自身的接口向网关中心注册。因为每一个RPC服务本身是在RPC注册中心维护的，具备负载均衡的能力。所以通常向网关中心注册的都是 RPC 的接口描述信息，不过网关中心可以在这个过程记录上 RPC 接口的总数以及IP信息。**这里暂时不需要，所以不提供注册**
